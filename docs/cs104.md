# `cs104` — IEC 60870-5-104 over TCP/IP

The `cs104` package implements the IEC 60870-5-104 transport (APCI) with the
full state machine: I/S/U frames, k/w flow-control windows, t₀–t₃ timeouts,
StartDT/StopDT activation and TestFR keep-alive. It offers:

- `cs104.Server` — a controlled station (slave/outstation) that listens for
  TCP connections from masters.
- `cs104.Client` — a controlling station (master) that connects to a server.
- `cs104.NewServerSpecial` — a controlled station that **dials out** to the
  master (reverse connection) instead of listening.

Default port: `cs104.Port` (2404), `cs104.PortSecure` (19998, TLS).
Default ASDU parameters: `asdu.ParamsWide` (COT 2 with originator address,
CA 2, IOA 3) — the IEC 104 standard layout.

## Server (controlled station)

```go
srv := cs104.NewServer(handler)              // handler: cs104.ServerHandlerInterface
srv.SetConfig(cs104.DefaultConfig())         // optional; invalid config falls back to defaults
srv.SetParams(asdu.ParamsWide)               // optional; must match the masters
srv.SetInfoObjTimeZone(time.UTC)             // time zone for CP24/CP56 tags
srv.SetTLSConfig(tlsCfg)                     // optional TLS
srv.LogMode(true)                            // enable logging (clog)
srv.SetOnConnectionHandler(func(c asdu.Connect) { ... })
srv.SetConnectionLostHandler(func(c asdu.Connect) { ... })

err := srv.ListenAndServer(":2404")          // blocks; returns the error that stopped it
// srv.Close() from another goroutine stops it
```

- Accepts any number of concurrent master connections
  (`srv.GetSessionsLen()` reports how many).
- `srv.Send(pack)` (and therefore the `asdu.*` helpers called with `srv` as
  the `asdu.Connect`) **broadcasts a copy to every connected, activated
  session** — this is how spontaneous data is published.
- Inside a handler, the `asdu.Connect` argument is the *single session* the
  request came from; replies through it go only to that master.

### ServerHandlerInterface

```go
type ServerHandlerInterface interface {
	InterrogationHandler(asdu.Connect, *asdu.ASDU, asdu.QualifierOfInterrogation) error
	CounterInterrogationHandler(asdu.Connect, *asdu.ASDU, asdu.QualifierCountCall) error
	ReadHandler(asdu.Connect, *asdu.ASDU, asdu.InfoObjAddr) error
	ClockSyncHandler(asdu.Connect, *asdu.ASDU, time.Time) error
	ResetProcessHandler(asdu.Connect, *asdu.ASDU, asdu.QualifierOfResetProcessCmd) error
	DelayAcquisitionHandler(asdu.Connect, *asdu.ASDU, uint16) error
	ASDUHandler(asdu.Connect, *asdu.ASDU) error          // everything else, incl. commands
	ASDUHandlerAll(asdu.Connect, *asdu.ASDU, int) error  // sees every ASDU first (int = server number)
}
```

Dispatch rules (per received type):

- `C_IC_NA_1`, `C_CI_NA_1`, `C_RD_NA_1`, `C_CS_NA_1`, `C_RP_NA_1`,
  `C_CD_NA_1` → the dedicated handler. The session **pre-validates** cause
  of transmission, common address and IOA and automatically replies
  `UnknownCOT`/`UnknownCA`/`UnknownIOA` mirrors on violations — your handler
  only sees valid requests.
- `C_TS_NA_1` (test command) is answered automatically with `ActivationCon`.
- Everything else — **including control commands C_SC/C_DC/C_RC/C_SE/C_BO
  and parameter commands** — goes to `ASDUHandler`. If `ASDUHandler`
  returns an error, the session replies with an `UnknownTypeID` mirror.
- `ASDUHandlerAll` is called for *every* inbound ASDU before dispatch
  (useful for logging/forwarding); its error does not stop dispatch.

### Typical interrogation handler

```go
func (h *myHandler) InterrogationHandler(c asdu.Connect, pack *asdu.ASDU, qoi asdu.QualifierOfInterrogation) error {
	if qoi != asdu.QOIStation {
		pack.Coa.IsNegative = true
		return pack.SendReplyMirror(c, asdu.ActivationCon)
	}
	_ = pack.SendReplyMirror(c, asdu.ActivationCon)
	// send the whole process image with the interrogation cause
	_ = asdu.Single(c, false, asdu.CauseOfTransmission{Cause: asdu.InterrogatedByStation},
		pack.CommonAddr, points...)
	_ = asdu.MeasuredValueFloat(c, false, asdu.CauseOfTransmission{Cause: asdu.InterrogatedByStation},
		pack.CommonAddr, measurements...)
	return pack.SendReplyMirror(c, asdu.ActivationTerm)
}
```

### Typical command handler

```go
func (h *myHandler) ASDUHandler(c asdu.Connect, pack *asdu.ASDU) error {
	switch pack.Type {
	case asdu.C_SC_NA_1, asdu.C_SC_TA_1:
		cmd := pack.GetSingleCmd()
		if cmd.Qoc.InSelect {
			// select phase: just confirm
			return pack.SendReplyMirror(c, asdu.ActivationCon)
		}
		ok := operate(cmd.Ioa, cmd.Value) // your I/O
		if !ok {
			pack.Coa.IsNegative = true
			return pack.SendReplyMirror(c, asdu.ActivationCon)
		}
		_ = pack.SendReplyMirror(c, asdu.ActivationCon)
		return pack.SendReplyMirror(c, asdu.ActivationTerm)
	}
	return nil // unknown types: return an error to auto-reply UnknownTypeID
}
```

## Client (controlling station)

```go
option := cs104.NewOption()
_ = option.AddRemoteServer("127.0.0.1:2404") // "tcp://host:port"; "tls://"/"ssl://"/"tcps://" for TLS
_ = option.SetConfig(cs104.DefaultConfig())
_ = option.SetParams(asdu.ParamsWide)
option.SetAutoReconnect(true).SetReconnectInterval(time.Minute)
option.SetTLSConfig(tlsCfg) // when using a tls:// URL

client := cs104.NewClient(handler, option)   // handler: cs104.ClientHandlerInterface
client.LogMode(true)
client.SetOnConnectHandler(func(c *cs104.Client) { c.SendStartDt() })
client.SetOnActivatedHandler(func(c *cs104.Client) {
	_ = c.InterrogationCmd(asdu.CauseOfTransmission{Cause: asdu.Activation}, 1, asdu.QOIStation)
})
client.SetOnDeactivatedHandler(func(c *cs104.Client) { ... })
client.SetConnectionLostHandler(func(c *cs104.Client) { ... })
client.SetConnectTimeoutHandler(func(c *cs104.Client) { ... })

_ = client.Start()   // returns immediately; connection managed in background
// ...
_ = client.Close()
```

**You must call `SendStartDt()`** after connecting — a 104 connection starts
in the STOPDT state and the server discards I-frames until StartDT is
confirmed. `client.IsActive()` reports the activation state;
`client.Send(...)` returns `ErrNotActive` before activation.

Command wrappers (all take `coa`, `ca`, plus specifics):
`InterrogationCmd`, `CounterInterrogationCmd`, `ReadCmd`,
`ClockSynchronizationCmd`, `ResetProcessCmd`, `DelayAcquireCommand`,
`TestCommand`. For control commands use the `asdu` helpers with the client
as the connection:

```go
_ = asdu.SingleCmd(client, asdu.C_SC_NA_1,
	asdu.CauseOfTransmission{Cause: asdu.Activation}, 1,
	asdu.SingleCommandInfo{Ioa: 6000, Value: true})
```

### ClientHandlerInterface

```go
type ClientHandlerInterface interface {
	InterrogationHandler(asdu.Connect, *asdu.ASDU) error         // C_IC confirmations (ActCon/ActTerm)
	CounterInterrogationHandler(asdu.Connect, *asdu.ASDU) error  // C_CI confirmations
	ReadHandler(asdu.Connect, *asdu.ASDU) error                  // C_RD confirmations
	TestCommandHandler(asdu.Connect, *asdu.ASDU) error           // C_TS confirmations
	ClockSyncHandler(asdu.Connect, *asdu.ASDU) error             // C_CS confirmations
	ResetProcessHandler(asdu.Connect, *asdu.ASDU) error          // C_RP confirmations
	DelayAcquisitionHandler(asdu.Connect, *asdu.ASDU) error      // C_CD confirmations
	ASDUHandler(asdu.Connect, *asdu.ASDU, *Server, int) error    // all data (M_*) and other types
	ASDUHandlerAll(asdu.Connect, *asdu.ASDU, *Server, int) error // sees every ASDU first
}
```

**Note the routing:** on the client side, the dedicated handlers receive the
*mirrored command confirmations* (the echoed C_* ASDUs with `ActivationCon` /
`ActivationTerm`). The actual **process data (M_SP, M_ME, M_IT, ...) —
including interrogation responses — arrives in `ASDUHandler`**. Decode it
there:

```go
func (h *cliHandler) ASDUHandler(c asdu.Connect, pack *asdu.ASDU, _ *cs104.Server, _ int) error {
	switch pack.Type {
	case asdu.M_SP_NA_1, asdu.M_SP_TB_1:
		for _, p := range pack.GetSinglePoint() { store(p.Ioa, p.Value, p.Qds, p.Time) }
	case asdu.M_ME_NC_1, asdu.M_ME_TF_1:
		for _, m := range pack.GetMeasuredValueFloat() { store(m.Ioa, m.Value, m.Qds, m.Time) }
	}
	return nil
}
```

The `*Server` and `int` parameters support data-forwarding gateways: use
`client.SetPairedServer(srv)` and `client.SetClientNumber(n)` to have the
paired server and a client index handed to your handler (both may be
nil/zero when unused).

## Reverse connection: `NewServerSpecial`

A controlled station that connects out to the master (useful across NAT):

```go
option := cs104.NewOption()
_ = option.AddRemoteServer("master.example.com:2404")
ss := cs104.NewServerSpecial(serverHandler, option) // ServerHandlerInterface!
ss.SetOnConnectHandler(func(c asdu.Connect) { ... })
ss.SetConnectionLostHandler(func(c asdu.Connect) { ... })
_ = ss.Start()
```

It behaves like a server session (answers interrogations, StartDT, etc.) but
initiates the TCP connection and auto-reconnects.

## Configuration (`cs104.Config`)

| Field | IEC name | Range | Default | Meaning |
|-------|----------|-------|---------|---------|
| `ConnectTimeout0` | t₀ | 1–255 s | 30 s | TCP connection establishment timeout |
| `SendUnAckLimitK` | k | 1–32767 | 12 | max outstanding (unacknowledged) I-frames; sending stops at k |
| `SendUnAckTimeout1` | t₁ | 1–255 s | 15 s | ack timeout; connection is closed when exceeded |
| `RecvUnAckLimitW` | w | 1–32767 | 8 | ack at latest after w received I-frames (keep w ≤ ⅔·k) |
| `RecvUnAckTimeout2` | t₂ | 1–255 s | 10 s | max delay before acknowledging (S-frame) |
| `IdleTimeout3` | t₃ | 1 s–48 h | 20 s | idle time before a TestFR keep-alive is sent |

Zero values are replaced by defaults in `Config.Valid()`. Both peers should
use compatible k/w/t values (the defaults interoperate with virtually all
implementations).

## Behavior notes

- Sequence numbers are 15-bit and wrap at 32767; the window bookkeeping
  handles wraparound.
- A received I-frame with an unexpected send sequence number, or an
  acknowledge outside the window, closes the connection (per standard);
  auto-reconnect (client) then recovers.
- `Send` is asynchronous: ASDUs are queued and packed into I-frames by the
  connection loop. When the queue is full it returns `ErrBufferFulled` —
  back off and retry rather than dropping silently.
- I-frames received while the connection is not activated (STOPDT) are
  discarded with a warning.

## Errors

| Error | Meaning |
|-------|---------|
| `cs104.ErrUseClosedConnection` | not connected |
| `cs104.ErrNotActive` | connected but StartDT not confirmed (client) |
| `cs104.ErrBufferFulled` | outbound ASDU queue full |
